# 4주차 - 암호화·인코딩·해시

## 1. 암호화

### 정의

암호화는 데이터를 다른 사람이 알아볼 수 없도록 바꾸는 것이다.  
암호화된 데이터는 올바른 키가 있으면 다시 원래 데이터로 복호화할 수 있다.

### 특징

- 데이터를 숨기는 것이 목적이다.
- 키가 필요하다.
- 복호화가 가능하다.

### 예시

- 원문 : `HELLO`
- 암호문 : `KHOOR`

카이사르 암호처럼 문자를 일정한 칸 수만큼 밀어서 암호문을 만들 수 있다.

---

## 2. 인코딩

### 정의

인코딩은 데이터를 정해진 규칙에 따라 다른 표현 방식으로 바꾸는 것이다.  
보안을 위한 것이 아니라 저장이나 전송을 쉽게 하기 위한 방식이다.

### 특징

- 데이터를 숨기는 목적이 아니다.
- 키가 필요 없다.
- 규칙만 알면 다시 원래 값으로 되돌릴 수 있다.

### 예시

- 원문 : `hello`
- Base64 인코딩 : `aGVsbG8=`

Base64는 암호처럼 보일 수 있지만 누구나 디코딩할 수 있기 때문에 암호화가 아니다.

---

## 3. 해시

### 정의

해시는 입력값을 고정된 길이의 값으로 바꾸는 것이다.  
암호화와 다르게 원래 값으로 되돌리는 것을 목적으로 하지 않는다.

### 특징

- 같은 입력값은 항상 같은 해시값이 나온다.
- 입력값이 조금만 바뀌어도 해시값이 크게 달라진다.
- 해시값으로 원래 입력값을 알아내기 어렵다.
- 데이터 검증이나 비밀번호 확인에 사용된다.

### 예시

- 입력값 : `hello`
- 해시값 예시 : `2cf24dba5fb0a...`

해시값은 사용하는 해시 알고리즘에 따라 달라진다.

---

# 암호화, 인코딩, 해시 비교

| 구분 | 목적 | 원래 값으로 되돌리기 | 키 필요 여부 | 예시 |
| --- | --- | --- | --- | --- |
| 암호화 | 내용을 숨김 | 가능 | 필요 | AES, RSA |
| 인코딩 | 표현 방식 변환 | 가능 | 필요 없음 | Base64, ASCII |
| 해시 | 검증 및 비교 | 불가능에 가까움 | 필요 없음 | SHA-256, MD5 |

---

# 해시의 기본 개념

## 1. 같은 입력은 같은 결과

해시는 같은 값을 입력하면 항상 같은 결과가 나온다.  
예를 들어 `hello`를 해시하면 매번 같은 해시값이 나온다.

## 2. 입력이 조금만 바뀌어도 결과가 달라짐

`hello`와 `Hello`는 비슷해 보이지만 서로 다른 값이다.  
그래서 해시값도 완전히 다르게 나온다.

## 3. 원래 값으로 되돌리는 용도가 아님

해시는 원래 데이터를 다시 꺼내기 위한 방식이 아니다.  
원문을 저장하지 않고도 같은 값인지 비교하기 위해 사용한다.

---

# 비밀번호를 복호화하지 않는 이유

## 1. 비밀번호 원문은 필요 없다

사이트는 사용자의 비밀번호를 직접 알 필요가 없다.  
로그인할 때는 입력한 비밀번호가 기존 비밀번호와 같은지만 확인하면 된다.

## 2. 복호화 가능한 저장 방식은 위험하다

비밀번호를 암호화해서 저장하면 키가 유출되었을 때 원문 비밀번호가 모두 노출될 수 있다.  
그래서 비밀번호는 복호화 가능한 형태로 저장하지 않는다.

## 3. 실제 로그인 방식

회원가입할 때 서버는 비밀번호 원문을 저장하지 않고 해시값을 저장한다.

로그인할 때는 사용자가 입력한 비밀번호를 다시 해시한 뒤, 저장된 해시값과 비교한다.  
두 값이 같으면 올바른 비밀번호로 판단한다.
